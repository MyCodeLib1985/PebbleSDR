Why Pebble?

I 'discovered' Software Defined Radio at the beginning of 2010 when I found myself between jobs and with extra time.  The experience of building my first SoftRock V9 and actually hearing something was like the first crackle from a crystal radio.  I wasn't quite sure everything was working, but it made noise!  I was hooked and soon wanted to learn about the software side of things.  I've got a background in software, but nothing in DSP. I looked at every application I could find and discovered that there was a lot of 'variety' in the algorithms and very little documentation.  I then started collecting and reading the standard DSP books (see references in pebblegpl.txt) but often wound up even more confused.  So I decided the only way to really learn and understand was to try to write something myself.  With that in mind, I came up with the following design goals:

  •  Small and simple.  There are plenty of full featured applications and I wanted something that didn't take up the entire screen and just did the basics. So you won't find precise S-Meters or DX grade features.
  • Works well with programs like fldigi.  This is part of the 'keep it small' principal.
  • Runs on Win7 - 64 (my base system), but has the potential to also run on Mac and Linux.  This lead to the decision to implement in C++ and the Qt cross platform framework
  • May or may not run on older OS's like XP.  There are plenty of great XP programs like Rocky.
  • Support multiple SDRs, or at least the ones that I've collected!
  • Help test new SDR builds and modifications.  Source should be well commented and source material for alorithms documented.

Current State

This is an early alpha build that has only limited testing on my Win7x64 system.  Other testers have reported success using Win7x32 and XP.

Modes

All the usual modes are supported, with the exception of DRM.  If you select "None" in the mode box, Pebble does not try to demodulate and just copies the raw IQ to the sound card output.  This allows it to be used with DREAM and FLDigi.  I've had good luck using DREAM in the IQ mode (startup with -c5).

Tuning

Tuning is still a work in process and I've been experimenting with different ideas. Here are the current options:

  • Click on the buttons above or below each digit to increment/decrement by that step.  Click and hold to tune continuously.  Note that there is a known problem with si570 based devices when the tuning increment is more than the 'smooth tune' increment.  You'll hear pops when this happens, try incrementing by 100k and your see what I mean.  I've been working to try to minimize this, but so for no joy.
  • Click on a digit to zero all the lower digits.
  • Direct entry: Type freq in hz in small box and hit enter
  • Note: Each SDR has a hi/lo frequency range defined in the .ini file for the device and the tuner will stop when it reaches that limit.  If you have a modified SDR that can do extended tuning, just change the ini file.
  • Mixer tuning (click on Mixer button) will tune within a +/-  range using the same controls.
  • Spectrum Tuning: Click anywhere is spectrum, LO or Mixer will change depending on mode.
  • Spectrum Tuning: Left/Right will increment by 1hz, Up/Down by 100hz.  This is configurable in sdr.ini.
  • Note: The Elektor SDR uses a different tuning system which steps in certain increments.  If you have one of these, try it and see if it works for you.
  • Note: If you are using a non-tunable device like the SoftRock Lite, set the LO to the device frequency, select mixer, and you can now tune and read frequency correctly.
  • Note: Different SDR devices can be selected which allows the frequency to be displayed correctly.  For example selecting the SoftRock Ensemble 2M VHF will allow you to enter VHF freq directly.
  • In CWL and CWU, a plus/minus 700hz offset is automatically applied so you can click directly on a signal in the waterfall.

Plot Display

The plot display as several operational modes, some testing modes, or can be turned off to save CPU.  There is a 'gain' slider on the right of the display that can be adjusted to help see weak signals.  The display will resize with the window, and will work even at full screen.

  • Spectrum: Two buffer average.  Gain is adjustable with slider on left.  Resizes with window
  • Waterfall: Gain is adjustable.  Resizes with window, but waterfall is blanked and starts over
  • IQ: This is a plot of the raw IQ and something I found useful for testing.  If you don't see two distinct plots, you may have a problem with stereo input.  Fun to look at when testing with a signal generator or WWV
  • Phase: Also raw IQ, but plotted in a phase graph.  Not sure how accurate or useful it is, but also fun to watch.
  • Off:
  • Post Mixer:  Shows the spectrum after the mixer, vs before mixer.  Used to test mixer and observe how spectrum changes as a result.
  • Post BandPass: Shows the spectrum after the selected bandpass filter.  I used this extensively to test and try different filter algorithms.  If you look at the code you may see option1, option2 etc

Settings - General

  • Receiver:  Allows you to select a different SDR device.  The goal eventually is to allow multiple devices to be plugged into USB and switch between them.  Switching to a new device will 'restart' Pebble so the device specific options can be applied.
  • Serial Number (Experimental): Softrocks can have an optional ID set using CFGSR or the Pebble SDR Options page.  This allows you to have multiple SoftRocks plugged in at the same time.  The default for this option is "Any" which will not check this serial number.  But if you set to a specific number, Pebble will only connect to a SoftRock with that same serial number.  This work well for me as long as the SoftRocks are on different frequencies.  If they are close I get signal interferance from the si570 and have not been able to isolate.
  • Startup Frequency: Startup with last frequency/mode, preset frequency/mode, or the device default.
  • Source/Output: Select available input/output devices.  Only MME devices are supported right now.

Settings - IQ

  • I/Q Order:  This allows you to swap I & Q, listen to only I or only Q.  The change happens immediately so you can try different settings, watch the spectrum display, without pressing SAVE.  Very helpful in testing sometimes.
  • IQ Gain adjust – Look at image signal in spectrum or waterfall and adjust gain and phase to null.  Click enable/disable when done to toggle between uncorrected and corrected.
  • IQ Phase adjust

Settings - SoftRock

  • SoftRock all: Displays version, serial # and current LO frequency
  • SoftRocks with ABPF: Allows you to manually selected a filter or automatic.  I found this very helpful in testing new SoftRock builds and experimenting with different antennas.
  • Serial Number: See usage in General Settings.  Allows you to select a specific SR among several that may be plugged in.

Settings - Elektor

Settings - SDR-IQ

  • RF Gain: Selectable attenuator 0db  to -30db  This changed in real time as you select options.
  • IF Gain: Selectable gain 0db to 24db.  Use combination of RF Gain and IF Gain to get correct spectrum display and prevent audio overload.  This is changed in real time as you select options.
  • Bandwidth: This is directly related to sample width and determine how wide the spectrum and waterfall display are.  Note that the SDR-IQ sends I/Q data directly through the USB port, so the sound card is only used for output.  Bandwidth can only be changed by toggling power off/on since it changes so much internal data.

Presets

The current preset functionality allows you to define as many stations and band as you like.  The easiest way to edit these is with a text editor or spreadsheet, just open presets.csv.  You can also edit them in the Pebble UI.

  • Click Presets to open presets window, click again to close (Note this won't collapse the window yet)
  • Click on any preset to select.
  • Click on the 'pencil' in the first column to delete or edit.  If editing, click on it again to complete.
  • Click New to add the current frequency/mode
  • Clck Save to write the presets.csv.  If you don't do this, any changes are lost.  So if you are making lots of changes in Pebble, save frequently!

DSP

DSP features are still a work-in-process but include;

  • Noise Blanker 1: Handles spikes, like lightning, by blanking output for several samples
  • Noise Blanker 2: Reduces average noise
  • Automatic Noise Filter
  • Low Pass filter (3k cutoff, after demod stage.  Testing)
  • AGC (currently defaults to off, but I may change that to slow so that AGC gain is active).  Normal AGC modes are supported, in addition to an AGC 'Gain' slider.

Misc

If you see 'over' or 'under' in the upper corners of the spectrum display, Pebble is not keeping up with incoming samples.  Try changing the sample rate or framesPerBuffer in Pebble.ini and see if that resolves the issue.

Tune to a strong WWV freq and switch the display to IQ mode.  If you don't see two distinct traces, you do not have stereo input.  Then switch display to Phase mode.  You should see a more-or-less round figure after you adjust display gain on right.

If you have an SDR-IQ, try switching to a 5k or 10k bandpass (slow sample rate) and look at IQ and phase plots for fun.

INI Files 

Pebble reads and saves program and device options as .ini files which are located in the same directory as pebble.exe.  These files are not included in the distribution, but are automatically created when the program runs.  To restore Pebble to initial defaults, just delete the relevant .ini files.  Program options are stored in pebble.ini and device options in device specific .ini files (softrock.ini, elektor.ini, etc).  If you need to edit a device .ini file which does not exist, select the device from General Settings, Power On (ignore any error messages), and Power Off.  The .ini file can then be edited.
